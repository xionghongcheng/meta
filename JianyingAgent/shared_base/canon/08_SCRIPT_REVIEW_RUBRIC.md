# 08 Script Review Rubric

## 使用方式

这份规则由 `Script Writer Agent` 和 `Script Evaluator Agent` 共享读取。

- `Script Writer Agent` 用它约束初稿方向，但不自判是否通过。
- `Script Evaluator Agent` 必须逐项打分，再给 review 结论。
- 工作顺序固定为：先写，再评，再决定是通过、小改还是重写。

## 一票否决项

出现以下任一情况，直接判定为 `重写`：

- 人物关系、人物能力或人物性格写错
- 明显违背账号世界观、场景规则或生活常识
- 为了煽情硬造冲突，失去真实感
- 过度消费老人病情、失能或脆弱处境
- 商业化内容先于真实问题出现
- 全文只有观点，没有可拍的动作、反应或细节

## 评分维度

总分 100 分。

### 1. 钩子（20 分）

重点看：

- 前几句能不能立住矛盾、反差或问题
- 开头是不是我家独有，而不是任何家庭都能套
- 有没有让人愿意继续看下去的具体信息

低分表现：

- 纯感慨开头
- 先讲道理，再讲故事
- 开头太空，没有人、动作和问题

### 2. 人物（20 分）

重点看：

- 四个人物的说话方式和行为是否区分清楚
- 每个人物有没有符合自己的位置和限制
- 人物之间的关系张力有没有写出来

低分表现：

- 谁说都一样
- 女朋友、爸、妈被写成工具人
- 人物反应不符合长期设定

### 3. 细节（20 分）

重点看：

- 有没有真实生活颗粒
- 有没有能拍到的动作、停顿、口头禅、表情、物件
- 细节是不是服务故事，而不是堆素材

低分表现：

- 只写结论，不写过程
- 只有抽象情绪，没有具体生活细节
- 细节泛化，像编出来的家庭故事

### 4. 推进（15 分）

重点看：

- 故事是不是一段一段往前走
- 转折是否自然
- 有没有重复表达同一个意思

低分表现：

- 中段塌陷
- 句子在打转
- 靠解释推进，而不是靠事件推进

### 5. 情绪（15 分）

重点看：

- 情绪是不是从事件和细节里自然长出来
- 有没有留白
- 结尾是否克制，有余味

低分表现：

- 过度抒情
- 强行上价值
- 情绪过满，像配文，不像生活

### 6. 商业化边界（10 分）

重点看：

- 是不是先有真实问题，再有解决方案
- 产品或品牌有没有破坏内容真实感
- 观众获得的是照护经验，而不是硬推销感

低分表现：

- 产品先出场
- 夸大效果
- 借父母脆弱处境卖货

## 判定阈值

- `85-100`：通过。结构成立，只需要微调句子。
- `70-84`：小改。主结构可保留，但要改弱段、废话和虚假情绪。
- `0-69`：重写。方向或结构本身有问题。
- 触发任一一票否决项：直接 `重写`，不看总分。

## Review 输出要求

`Script Evaluator Agent` 输出固定包含以下部分：

```text
总评：通过 / 小改 / 重写
总分：XX/100

维度评分：
- 钩子：XX/20
- 人物：XX/20
- 细节：XX/20
- 推进：XX/15
- 情绪：XX/15
- 商业化边界：XX/10

必须修改：
- 只写会影响过稿的关键问题

可优化：
- 只写锦上添花的问题
```

## Review 原则

- 先指出硬伤，再说优化项。
- 必须引用具体句子或具体段落问题，不能只给抽象评价。
- 不替写整篇，只给修改方向和判断依据。
- 如果稿子的问题出在选题本身，要明确指出不是文笔问题，而是题不成立。
