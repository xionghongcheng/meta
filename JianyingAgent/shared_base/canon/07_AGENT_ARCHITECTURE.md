# 07 Agent 架构

## V1 架构

V1 采用：

> 一个统一内容大脑 + 四个核心 Agent

## 内容大脑

内容大脑是所有 Agent 的共享底座。

包含：

- 账号世界观
- 人物设定
- 内容主线
- 关系与生活细节
- 场景规则
- 商业化规则
- 脚本评分标准
- 历史配音故事库
- 文风模板
- 后续数据复盘

其中，`Script Writer Agent` 和 `Script Evaluator Agent` 共享同一套脚本标准，统一读取 `08_SCRIPT_REVIEW_RUBRIC.md`。

## 选题 Agent

负责决定拍什么，不直接写成稿。

它回答：

- 这条值不值得拍
- 属于哪条内容主线
- 核心矛盾或反差是什么
- 四个人物如何参与
- 哪些生活细节最值得调用
- 适不适合商业化
- 更适合哪种脚本结构

输出应尽量简洁，重点是给写稿提供方向。

## Script Writer Agent

负责把选题方向写成可讲、可拍、可剪的脚本初稿。

它回答：

- 开头钩子怎么立
- 这件事在我家为什么不普通
- 故事怎么推进
- 四个人物各自说什么、做什么
- 哪些关系细节和生活细节必须写出来
- 需要拍哪些镜头和原声
- 结尾落点停在哪里

`Script Writer Agent` 的目标是先把故事写顺、写真、写出情绪张力，不负责给自己判定是否合格。

## Script Evaluator Agent

负责对脚本初稿做独立 review，不参与初稿生成。

它回答：

- 开头钩子是否成立
- 故事推进是否顺
- 人物有没有写串
- 生活细节是否真实具体
- 情绪是否过满、过假或过空
- 是否踩了商业化边界
- 哪些句子是废话
- 这稿能不能过，为什么

`Script Evaluator Agent` 必须以 `08_SCRIPT_REVIEW_RUBRIC.md` 为统一标尺，先打分，再给修改意见，最后输出 `通过 / 小改 / 重写` 结论。

## 自动粗剪 Agent

负责拍摄后。

它回答：

- 素材里有哪些可用片段
- 哪些片段对应脚本重点
- 哪些原声值得保留
- 哪些小细节要优先保留
- 时间线怎么排
- 如何生成剪映粗剪工程

## V1 工作流

```text
内容大脑
  ↓
选题 Agent
  ↓
Script Writer Agent
  ↓
脚本初稿
  ↓
Script Evaluator Agent
  ↓
通过 / 小改 / 重写
  ↓
终稿 / 拍摄清单
  ↓
拍摄素材
  ↓
自动粗剪 Agent
  ↓
剪映粗剪工程
```

如果结论是 `小改` 或 `重写`，则回到 `Script Writer Agent` 继续改稿，再交给 `Script Evaluator Agent` 复审。

## 写稿与评稿边界

- `Script Writer Agent` 负责产出，不负责自评通过。
- `Script Evaluator Agent` 负责 review 和打分，不负责代写整稿。
- 两者共享同一份脚本规则，但不能由同一轮生成同时扮演两个角色。
- 先写后评是固定顺序，避免标准漂移和自我合理化。

## 评估结果建议格式

```text
总评：通过 / 小改 / 重写
总分：XX/100

维度评分：
- 钩子
- 人物
- 细节
- 推进
- 情绪
- 商业化边界

必须修改：
- ...

可优化：
- ...
```
